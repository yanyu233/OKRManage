## 1. 文档说明

- 文档名称：OKR 目标管理界面功能需求细化
- 文档版本：v1.0
- 编写日期：2026-04-01
- 设计依据：
  - 已有基础需求文档 `docs/okr-requirements.md`
  - 目标管理列表页示例
  - 新建目标弹窗示例
- 文档目的：基于现有界面示例，进一步细化目标管理模块的页面功能、字段规则、交互逻辑与业务规则，用于原型设计、研发拆解和测试验收。

## 2. 界面理解与设计原则

从示例界面看，当前系统更接近“目标管理 + 考核评分”一体化模型，而不是纯粹的标准 OKR 文本管理。系统需要同时支持：

1. 目标按树形层级管理，支持父子目标逐级拆解。
2. 目标下配置关键结果 KR，KR 作为最小执行与评分单元。
3. 目标既要体现进度，也要体现分值、评分和总结。
4. 管理者需要查看本人、关注对象、全部目标和下属目标。
5. 用户需要在一个轻量表单中完成目标新建、保存草稿和正式提交。

本次功能设计遵循以下原则：

1. 先保障“目标创建 - 跟踪 - 评分 - 汇总”主流程闭环。
2. 界面尽量保留树形结构，便于展示上下级承接关系。
3. 列表页兼顾执行视角与管理视角，支持快速筛选和批量查看。
4. 表单字段尽量清晰，减少用户填报负担。

## 3. 核心业务对象

### 3.1 目标对象层级

系统中的目标对象建议分为三层：

1. Objective：主目标，对应业务目标、部门目标、个人目标等。
2. Sub-Objective：子目标，用于对主目标继续拆解。
3. Key Result：关键结果，作为可衡量、可跟踪、可评分的最小单元。

建议支持如下层级关系：

- 目标可包含子目标
- 子目标可继续包含子目标
- 任意叶子目标下可包含一个或多个 KR

为控制复杂度，建议首期约束：

1. Objective 最大嵌套层级不超过 3 级。
2. KR 只能挂在最末级目标下，不允许 KR 下再挂子节点。

### 3.2 核心字段

目标主对象建议包含以下核心字段：

| 字段 | 说明 |
| --- | --- |
| 目标 ID | 系统唯一标识 |
| 目标编码 | 如 O1、O11、O121，按树形层级自动生成 |
| 目标名称 | 用户输入，列表主展示字段 |
| 目标类型 | 如公司、部门、团队、个人、项目 |
| 所属部门 | 目标归属组织 |
| 负责人 | 目标第一责任人 |
| 周期 | 所属考核周期，如 2026 年一季度 |
| 父目标 | 当前目标对齐或隶属的父目标 |
| 进度 | 当前完成百分比 |
| 分值 | 目标或 KR 权重 / 分值 |
| 评分 | 周期结束后的评价结果 |
| 状态 | 草稿、进行中、已完成等 |
| 目标总结 | 周期结束后的总结内容 |

KR 对象建议包含以下核心字段：

| 字段 | 说明 |
| --- | --- |
| KR ID | 系统唯一标识 |
| KR 编码 | 如 KR1、KR2 |
| KR 名称 | 关键结果名称 |
| 所属目标 | 关联的目标 ID |
| 衡量方式 | 数值型、百分比型、里程碑型、评分型 |
| 起始值 | 可选 |
| 目标值 | 必填，取决于衡量方式 |
| 当前值 | 默认 0 或空 |
| 分值 | KR 对应分值 |
| 负责人 | 默认继承目标负责人，可调整 |
| 状态 | 未开始、进行中、已完成、滞后 |
| 评分 | 周期结束时填写 |

## 4. 功能模块设计

## 4.1 目标管理首页

### 4.1.1 页面目标

目标管理首页用于集中展示当前用户可见范围内的目标数据，并支持查看、筛选、树形展开、进度跟踪和快速新建目标。

### 4.1.2 页面组成

依据示例页面，首页建议包含以下区域：

1. 左侧导航区
2. 顶部查询与快捷操作区
3. 中央目标树列表区
4. 右上角新建目标入口
5. 周期切换入口
6. 分页与统计区

### 4.1.3 左侧导航功能需求

| 编号 | 功能点 | 需求说明 | 优先级 |
| --- | --- | --- | --- |
| UIFR-001 | 我负责的 | 默认视图，展示当前用户为负责人的目标 | P0 |
| UIFR-002 | 我关注的 | 展示当前用户标记关注的目标 | P1 |
| UIFR-003 | 全部目标 | 展示当前用户有权限查看的全部目标 | P0 |
| UIFR-004 | 目标关系树 | 以独立树图或结构化列表展示目标承接关系 | P1 |
| UIFR-005 | 草稿箱 | 展示当前用户保存但未提交的目标草稿 | P0 |
| UIFR-006 | 我的下属 | 支持按下属成员切换查看其目标 | P1 |

### 4.1.4 顶部筛选与检索需求

| 编号 | 功能点 | 需求说明 | 优先级 |
| --- | --- | --- | --- |
| UIFR-010 | 全局搜索 | 支持按目标名称、负责人、部门关键字搜索 | P0 |
| UIFR-011 | 周期切换 | 支持切换目标周期，如季度、年度 | P0 |
| UIFR-012 | 状态筛选 | 支持按草稿、进行中、已完成、已关闭筛选 | P0 |
| UIFR-013 | 类型筛选 | 支持按公司、部门、团队、个人、项目筛选 | P0 |
| UIFR-014 | 快速统计 | 显示当前筛选结果下的目标数量 | P1 |
| UIFR-015 | 清空筛选 | 一键恢复默认筛选条件 | P1 |

### 4.1.5 目标树列表需求

列表区应支持树形结构展示，字段与交互建议如下：

| 列名 | 需求说明 |
| --- | --- |
| 目标 | 展示编码 + 名称，支持展开/收起子节点 |
| 进度 | 以进度条 + 百分比方式展示 |
| 类型 | 展示目标类型，如部门、个人、项目 |
| 部门/用户组 | 展示归属部门或用户组 |
| 分值 | 展示当前目标配置的分值 |
| 评分 | 展示最终评分或当前评分 |
| 目标总结 | 展示是否已填写总结，可点击查看详情 |

树形交互要求：

1. 支持逐级展开和收起。
2. 支持记忆上次展开状态。
3. 非叶子节点的进度、分值、评分默认由下级汇总展示。
4. KR 节点使用单独标识，与 Objective 节点区分显示。
5. 点击目标名称进入目标详情页。

### 4.1.6 列表数据规则

1. 根节点默认为当前查询范围内的一级目标。
2. 子目标按编码顺序展示。
3. KR 节点显示在所属目标下方，不与目标节点混排到同一层。
4. 若某目标无子级但有 KR，则展开后显示 KR 列表。
5. 无数据时展示空状态，并提示用户新建目标。

### 4.1.7 行级操作需求

建议在列表行悬停或右侧操作区提供以下能力：

| 编号 | 功能点 | 需求说明 | 优先级 |
| --- | --- | --- | --- |
| UIFR-020 | 查看详情 | 查看目标完整信息、进度、评论、历史记录 | P0 |
| UIFR-021 | 编辑目标 | 草稿或可编辑状态下允许编辑 | P0 |
| UIFR-022 | 新增子目标 | 在当前目标下快速创建子目标 | P0 |
| UIFR-023 | 新增 KR | 在当前目标下快速新增 KR | P0 |
| UIFR-024 | 关注/取消关注 | 用户可收藏关注目标 | P1 |
| UIFR-025 | 删除草稿 | 草稿状态可删除 | P0 |
| UIFR-026 | 提交审批 | 草稿目标可提交 | P0 |
| UIFR-027 | 更新进度 | 对进行中 KR 或叶子目标更新进度 | P0 |
| UIFR-028 | 填写总结 | 周期结束后填写目标总结 | P1 |

### 4.1.8 草稿箱需求

草稿箱需满足以下要求：

1. 保存为草稿的目标进入草稿箱。
2. 草稿按最近更新时间倒序展示。
3. 草稿可继续编辑、复制、删除、提交。
4. 草稿不进入正式统计和评分范围。

### 4.1.9 我的下属需求

1. 左侧展示当前管理者可查看的下属成员列表。
2. 支持按单个下属查看其负责目标。
3. 支持查看下属目标的进度、分值、评分和总结。
4. 若管理者具备权限，可对下属目标进行评论、审批和评分。

## 4.2 新建目标弹窗

### 4.2.1 页面目标

新建目标弹窗用于快速创建目标，并在同一表单中完成基础信息填写、父目标选择和 KR 配置。

### 4.2.2 表单字段设计

根据示例界面，新建目标弹窗建议字段如下：

| 字段 | 控件类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 目标名称 | 单行文本框 | 是 | 输入目标名称，限制 100 字内 |
| 目标类型 | 下拉选择 | 是 | 如公司、部门、团队、个人、项目 |
| 目标所属部门 | 组织选择器 | 是 | 选择归属部门，默认当前用户所在部门 |
| 负责人 | 人员选择器 | 是 | 默认当前用户，可按权限选择他人 |
| 目标周期 | 下拉选择 | 是 | 仅可选择当前开放周期 |
| 父目标 | 目标选择器 | 否 | 选择后自动建立树形关系 |
| 关键结果 | 动态表单区 | 否 | 可添加多个 KR |

### 4.2.3 字段交互规则

1. 目标类型切换后，父目标候选列表应自动按同周期、同权限范围过滤。
2. 所属部门变更后，负责人候选列表应优先显示该部门成员。
3. 负责人默认回填当前登录人。
4. 目标周期默认取当前页面已选周期。
5. 父目标为可选项，顶层目标允许不选父目标。
6. 若从“新增子目标”入口进入，则父目标自动带入且不可为空。

### 4.2.4 关键结果 KR 配置需求

点击“添加关键结果”后，建议新增一组 KR 编辑项，每组 KR 包含：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| KR 名称 | 是 | 关键结果名称 |
| 衡量方式 | 是 | 数值型、百分比型、里程碑型、评分型 |
| 目标值 | 是 | 根据衡量方式输入 |
| 分值 | 是 | 建议输入整数或一位小数 |
| 负责人 | 否 | 默认继承目标负责人 |
| 说明 | 否 | 补充 KR 的评价标准或口径 |

KR 区域交互要求：

1. 支持新增多个 KR。
2. 支持删除未保存的 KR。
3. 支持拖拽或上下移动 KR 顺序。
4. 若已新增 KR，则提交前需至少保证一个 KR 填写完整。
5. 若产品要求“所有目标必须带 KR”，则可在系统参数中开启强校验。

### 4.2.5 按钮行为定义

| 按钮 | 需求说明 |
| --- | --- |
| 取消 | 关闭弹窗，不保存变更 |
| 保存为草稿 | 保存当前内容，可缺少部分非关键字段，但必须具备基本标识信息 |
| 确定 | 正式保存目标；若启用审批流程，则进入待审批状态 |

### 4.2.6 表单校验规则

1. 目标名称不能为空，且不能全为空格。
2. 目标名称长度建议不超过 100 字。
3. 目标类型不能为空。
4. 所属部门不能为空。
5. 负责人不能为空。
6. 目标周期不能为空。
7. 父目标若已选择，则必须与当前目标周期一致。
8. 父目标若已选择，则当前目标所属部门和负责人需满足权限范围。
9. KR 分值应大于 0。
10. 若录入多个 KR，KR 总分值建议等于当前目标分值；若未设置目标分值，则由 KR 自动汇总。
11. 不允许选择自己或自己的下级目标作为父目标，避免循环引用。

### 4.2.7 保存逻辑

1. 保存草稿时，系统生成临时目标编码或草稿编号。
2. 正式提交时，系统生成正式目标编码。
3. 如果选择父目标，则编码按父级层级自动派生。
4. 若未填写 KR，系统允许先保存目标主体，后续补充 KR。

## 4.3 目标详情与维护

虽然示例中未展示详情页，但基于列表操作，目标详情页建议支持以下模块：

| 编号 | 功能点 | 需求说明 | 优先级 |
| --- | --- | --- | --- |
| UIFR-030 | 基本信息 | 展示目标名称、编码、负责人、周期、父目标、状态 | P0 |
| UIFR-031 | 结构视图 | 展示子目标与 KR 结构 | P0 |
| UIFR-032 | 进度记录 | 展示历次进度更新日志 | P0 |
| UIFR-033 | 评论协同 | 支持上级和协同人评论 | P1 |
| UIFR-034 | 附件管理 | 支持上传证明材料或复盘附件 | P1 |
| UIFR-035 | 评分总结 | 展示自评、上级评分和总结 | P0 |
| UIFR-036 | 操作历史 | 展示创建、编辑、审批、评分记录 | P1 |

## 4.4 进度与评分规则

### 4.4.1 进度计算规则

1. KR 为最小进度维护单元。
2. 数值型 KR 按 `当前值 / 目标值` 自动计算进度，最大不超过 100%。
3. 百分比型 KR 直接按录入百分比展示。
4. 里程碑型 KR 按节点完成情况转换为进度。
5. 父目标进度默认按子节点分值加权汇总。

### 4.4.2 分值规则

1. 目标分值用于体现该目标在上级目标中的权重。
2. 若当前目标存在子目标，则子目标分值之和应等于当前目标分值。
3. 若当前目标仅存在 KR，则 KR 分值之和应等于当前目标分值。
4. 系统应支持开启“允许不严格等于”的配置，用于过渡期管理。

### 4.4.3 评分规则

1. KR 评分为叶子节点评分。
2. 子目标评分按下级 KR 或子目标评分加权汇总。
3. 目标评分保留两位小数。
4. 评分状态建议分为未评分、已自评、已上级评分、已确认。
5. 页面列表中的“评分”列默认展示最终评分；未评分时显示“-”或“未完成”。

### 4.4.4 目标总结规则

1. 每个目标支持填写目标总结。
2. 目标总结包括完成情况、问题原因、改进建议。
3. 周期结束后方可填写正式总结，进行中阶段可填写阶段备注。
4. 汇总页应支持查看目标总结的填写状态。

## 4.5 编码与层级规则

依据示例中的 O1、O11、O121、KR1 展示方式，建议采用如下规则：

1. 一级目标编码格式：`O1`、`O2`、`O3`
2. 二级目标编码格式：继承父目标编号后追加序号，如 `O11`、`O12`
3. 三级目标编码格式：如 `O121`、`O122`
4. KR 编码在目标内局部递增，如 `KR1`、`KR2`
5. 编码由系统自动生成，用户不可手工修改
6. 删除中间节点后，不回收历史编码

## 4.6 状态流转设计

### 4.6.1 目标状态

建议目标状态包含：

- 草稿
- 待审批
- 已生效
- 执行中
- 已完成
- 已关闭
- 已归档

### 4.6.2 状态流转规则

1. 新建目标后点击“保存为草稿”，状态为草稿。
2. 新建目标后点击“确定”，若无审批则直接生效，若有审批则进入待审批。
3. 审批通过后状态变为已生效或执行中。
4. 周期开始后自动转为执行中。
5. 所有叶子 KR 完成后，目标可自动标记为已完成。
6. 周期结束后，未归档前可进入总结和评分阶段。
7. 归档后仅允许查看，不允许编辑。

## 4.7 权限与数据范围

### 4.7.1 创建权限

1. 普通员工可创建个人目标。
2. 部门负责人可创建部门目标。
3. 组织管理员可创建公司级或跨部门目标。
4. 是否允许为他人创建目标，可由权限配置控制。

### 4.7.2 查看权限

1. 用户默认查看本人负责的目标。
2. 用户可查看自己关注的目标。
3. 管理者可查看下属目标。
4. 管理员可查看全部目标。

### 4.7.3 编辑权限

1. 草稿状态下，创建人和授权管理者可编辑。
2. 已生效状态下，关键字段修改可触发再审批。
3. 已归档目标不可编辑。

## 6. 验收场景

### 6.1 新建目标

1. 用户进入目标首页，点击“新建目标”。
2. 填写目标名称、目标类型、所属部门、负责人、目标周期。
3. 选择父目标并新增 1 个以上 KR。
4. 点击“确定”后，目标成功出现在目标树中。
5. 若点击“保存为草稿”，目标出现在草稿箱中。

### 6.2 目标树查看

1. 用户在“我负责的”中看到自己负责的一级目标。
2. 展开目标后可查看子目标和 KR。
3. 进度、分值、评分信息展示正确。

### 6.3 管理者查看下属目标

1. 管理者进入“我的下属”。
2. 切换至某成员后可查看其目标树。
3. 可查看目标详情、进度和评分信息。

### 6.4 周期切换

1. 用户切换到其他季度。
2. 列表只展示对应周期的数据。
3. 新建目标弹窗中的默认周期同步变化。

## 7. 默认假设

1. 当前界面中的“目标”与“OKR”统一按目标管理体系理解。
2. 分值用于权重和考核汇总。
3. 评分在周期末或节点性考核时填写。
4. 目标树中的 O 和 KR 编码均由系统自动生成。
5. 首期默认一条目标可有多个子目标或多个 KR，但不建议同时混用过深层级。
