# Route C C7 负责人全量查看、范围评分与批量评分设计

**目标**

在 Route C 新架构里补齐负责人视图和评分权限：

- 科室负责人/小组负责人都可以查看所有员工的 OKR
- 评分权限只对本科室/本小组生效，取并集
- 评分排名对所有科室负责人/小组负责人开放全量数据
- 多角色用户在顶部和左侧菜单中显示完整角色能力
- 评分工作台支持批量评分，并提供按科室/小组/员工筛选与全选快捷操作
- 模板导入的目标固定排在季度目标最前面，依次占用 `O1/O2/O3`

## 业务规则

### 负责人查看与评分边界

- 所有 `section-leader` 与 `group-leader` 都能查看所有员工的季度 OKR 和材料
- 评分权限按绑定范围判断：
  - `section-leader` 可评分本科室员工
  - `group-leader` 可评分本小组员工
  - 同时具有两种绑定时，评分范围取并集
- 对不在评分范围内的员工：
  - 工作台允许查看
  - 单条评分禁用
  - 批量评分时自动跳过并返回跳过数量

### 多角色展示

- 顶部“当前角色”显示当前用户实际拥有的角色组合，而不是仅显示 `activeRole`
- 角色顺序固定为：
  - `系统管理员`
  - `科室负责人`
  - `小组负责人`
  - `员工`
- 左侧菜单按角色能力分组展示
- 如果用户同时是 `员工 + 小组负责人`，菜单中显示三项能力：
  - `我的 OKR`
  - `评分工作台`
  - `评分排名`

### 批量评分

- 评分工作台新增“批量评分”入口
- 批量评分弹窗支持：
  - 科室筛选
  - 小组筛选
  - 员工多选
  - 全选员工
  - 全选目标
  - 全选关键结果
  - 全选（去除模板目标）
- 批量评分的实际写入单位仍然是关键结果
- 默认不覆盖已有评分；如需覆盖，由显式开关控制

### 模板目标排序

- 一模板一目标，不合并
- 模板导入目标固定排在季度目标最前
- 多个模板目标依次占用 `O1/O2/O3...`
- 普通目标顺延到模板目标之后
- “全选（去除模板目标）”会排除所有模板导入目标及其下关键结果

## 技术落点

- 后端：
  - `leader` 模块拆分“可见范围”和“可评分范围”
  - 新增批量评分接口
  - `employee` 模块导入模板后重排季度目标编码
- 前端：
  - `AppShell` 与 `routing` 补齐多角色组合展示
  - `LeaderWorkbenchPage` 增加批量评分弹窗和只读/可评分态
  - 负责人类型定义补齐 `canScore` 等字段

## 验证重点

- 负责人可查看所有员工
- 对超出评分范围的员工，评分接口返回拒绝
- 批量评分只更新有权限的关键结果
- 模板导入后目标编码顺序正确
- 多角色用户菜单与顶部角色文本正确
