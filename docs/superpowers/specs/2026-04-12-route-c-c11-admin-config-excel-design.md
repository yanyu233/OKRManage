# Route C C11 系统配置 Excel 导入导出设计

**日期**

2026-04-12

**目标**

为 Route C 系统管理员配置页增加一个总 Excel 工作簿导入导出能力，覆盖全部配置模块，并支持“只修改其中几张 sheet 后局部导入”，同时保持现有页面保存、校验和负责人权限逻辑不变。

## 范围

### 本次包含

- 系统配置页顶部新增 `导出 Excel` / `导入 Excel`
- 导出一个总工作簿，包含多张 sheet
- 导入支持局部 sheet 更新
- 导入结果自动刷新系统配置页草稿和服务端快照
- 模块级校验失败时整次导入失败
- 名称 + 隐藏 ID 的混合关联模式
- 回归测试与本地验证

### 本次不包含

- Excel 差异预览向导
- 模板化批量模板下载
- 多语言工作簿
- CSV 导入导出

## 设计结论

采用“**一个总 Excel，多张 sheet，支持局部导入**”。

导出时生成一个统一工作簿；导入时只处理“存在且有有效数据”的 sheet，未出现或空白的 sheet 保持系统当前数据不变。导入前以当前 bootstrap 为基线，按模块覆盖，再复用现有 `saveBootstrap` 校验和持久化路径。

## 工作簿结构

固定包含以下 sheet：

- `说明`
- `部门`
- `科室`

其中：

- `说明` 只做导入规则说明，不参与导入
- 其它 sheet 与系统配置页五个模块一一对应

## 导出规则

- 工作簿由服务端生成，前端只负责下载
- 所有 sheet 都按当前系统配置快照生成
- 隐藏 ID 列写入，但默认隐藏
- `说明` sheet 明确写出：
  - 可以只修改部分 sheet
  - 空 sheet 不会覆盖系统数据
  - 名称优先用于人工维护，ID 用于稳定匹配
  - 名额超过组内人数会整次导入失败

## 导入规则

### 1. 局部导入

- 只处理存在且有有效数据行的 sheet
- 缺失 sheet 或只有表头的 sheet 视为“未参与本次导入”
- 导入前先读取当前 bootstrap
- 对参与导入的模块做覆盖，不参与导入的模块保持当前值

### 2. 关联匹配

优先级：

1. 先按隐藏 ID 匹配
2. 若 ID 缺失，再按名称匹配
3. 若两者都匹配不到，则按新增处理

### 3. 删除语义

本次导入先不支持“通过删掉 Excel 行来删除系统数据”。

原因：

- 局部导入和删除语义组合后容易误删
- 当前系统配置页本身已有可视化删除

所以本次规则是：

- 导入可新增
- 导入可修改
- 导入不负责删除

### 4. 校验与失败策略

导入过程按整次事务处理：

- 任意参与导入的模块校验失败，整次导入失败
- 不做部分成功落库

继续复用现有规则：

- 评价组名额总数不能超过组内员工人数
- 模板目标必须有部门
- 模板目标必须至少有一个关键结果
- 模板关键结果必须有编号、名称、非负整数分值
- 部门、评价组、模板名称等唯一性规则继续生效

## API 设计

新增：

- `GET /api/admin/org/bootstrap/excel`
  - 返回 `.xlsx` 文件流
- `POST /api/admin/org/bootstrap/excel`
  - 接收上传文件
  - 解析工作簿
  - 合并到当前 bootstrap
  - 复用现有保存链路
  - 返回最新 bootstrap 和本次实际导入的模块列表

## 前端交互

系统配置页顶部按钮区新增：

- `导出 Excel`
- `导入 Excel`

交互：

- 点击导出直接下载当前配置工作簿
- 点击导入打开文件选择，只接受 `.xlsx`
- 导入成功后：
  - 提示成功
  - 明确显示本次更新了哪些模块
  - 刷新当前草稿和统计卡
- 导入失败后：
  - 保持当前页面状态
  - 显示后端返回的具体校验错误

## 测试与验收

### 后端

- 导出接口返回可解析工作簿
- 局部导入评价组名额只更新名额，不影响模板目标
- 局部导入模板目标只更新模板，不影响角色
- 名额超限时整次导入失败
- 模板关键结果非法时整次导入失败

### 前端

- 系统配置页可触发导出
- 导入按钮只接受 `.xlsx`
- 导入成功后展示导入模块提示
- 导入失败后展示错误信息

### 手工验收

- 导出当前配置
- 只改 `评价组名额` sheet 再导入
- 只改 `模板目标 / 模板关键结果` sheet 再导入
- 用错误名额导入并确认系统数据未被污染
